Title
Web Application Architecture 101
Date
8/25/2015

Two-tier

The two-tiered monolith web application.

Figure: Two-tier Web Application (Single Instance)

  • Usually the default style
  • Simple and easy to setup

Figure: Two-tier Web Application (Ecosystem)

As these proliferate they tend to integrate via a shared DB.

Three-tier

Three-tier architecure introduced a service layer between the web layer and the DB.

Figure: Three-tier Web Application (Single Instance)

The service layer encapsulates business logic and data access and provides an API for web layer.

Figure: Three-tier Web Application (Ecosystem)

Commonly the service layer is viewed as a single canonical thing (monolith). You could also break it into x pieces and distribute them (microservices).

Figure: Three-tier Web Application (Scalability Benefit)

Once benefit of the extra work for of creating the service tier is that you can scale each layer independently which can give you great deployment options in terms of location based caching and distribution.
